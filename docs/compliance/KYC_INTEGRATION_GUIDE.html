KYC (Know Your Customer) Integration Guide

📋 Overview

The KYC verification system is now fully integrated into the B2B Admin platform. It provides:

  • User Interface: Multi-step KYC submission form
  • Status Tracking: Real-time verification status
  • Admin Dashboard: Review and manage KYC submissions
  • Backend Service: Dedicated KYC microservice

🗺️ Navigation & Routes

User Routes (Public)

Route Component Description
/kyc/submit KYCSubmissionPage Submit KYC verification with multi-step form
/kyc/status KYCStatusPage Check current verification status

Admin Routes (Protected - Requires Auth)

Route Component Description
/admin/kyc KYCAdminDashboard Review pending KYC submissions and manage approvals
✓ Navigation Menu: KYC Verification is now available in the sidebar under User Management section with submenu items for admin submissions and user status check.

🔌 Backend Services

KYC Service Configuration

Variable Value Purpose
KYC_SERVICE_PORT 3011 Service port (configured in .env)
KYC_SERVICE_URL http://localhost:3011 External reference URL
JWT_SECRET Required Token verification key
DATABASE_URL_CORE PostgreSQL URI Database connection

Starting the KYC Service

Run one of the following commands:

# Development (with hot-reload) npm run dev --workspace=@tripalfa/kyc-service # Production build npm run build --workspace=@tripalfa/kyc-service npm start --workspace=@tripalfa/kyc-service

Health Check

GET http://localhost:3011/health

API Documentation

GET http://localhost:3011/kyc/api-docs (Swagger UI)

🔐 API Endpoints

User Endpoints (Authentication Required)

POST /auth/b2b-register/kyc-submit

Submit KYC verification information

Request Body:
{
  "firstName": "John",
  "lastName": "Doe",
  "dateOfBirth": "1990-01-01",
  "nationality": "US",
  "address": "123 Main St",
  "documentType": "passport",
  "documentNumber": "A12345678"
}
POST /auth/b2b-register/kyc-upload

Upload KYC documents (front, back, selfie, proof of address)

GET /auth/b2b-register/kyc-status

Get current KYC verification status

Admin Endpoints (Authentication + Admin Role Required)

GET /api/admin/kyc/pending?page=1&limit=20

Get list of pending KYC submissions

GET /api/admin/kyc/:verificationId

Get detailed KYC verification information

POST /api/admin/kyc/:verificationId/approve

Approve KYC verification

POST /api/admin/kyc/:verificationId/reject

Reject KYC verification with reason

POST /api/admin/kyc/:verificationId/request-documents

Request additional documents from user

GET /api/admin/kyc/stats

Get KYC statistics (pending, approved, rejected counts)

GET /api/admin/kyc/export?format=csv&status=all

Export KYC data (CSV or JSON)

🚀 Quick Start

For Users:

  1. Navigate to /kyc/submit
  2. Fill out the 4-step form with personal information
  3. Upload required documents (front, back, selfie, proof of address)
  4. Review and submit
  5. Check status at /kyc/status (auto-refreshes every 30 seconds)

For Admins:

  1. Navigate to /admin/kyc from the sidebar
  2. View statistics (pending, approved, rejected)
  3. Review pending submissions in the table
  4. Click "View" to see detailed document information
  5. Approve, reject, or request additional documents
  6. Export data as CSV or JSON

🔧 Environment Setup

# .env file requirements
KYC_SERVICE_PORT=3011
KYC_SERVICE_URL=http://localhost:3011
JWT_SECRET=your-secret-key
JWT_ISSUER=tripalfa
ENABLE_KYC_VERIFICATION=true
DATABASE_URL_CORE=postgresql://user:password@localhost:5432/tripalfa_core
INTERNAL_API_KEY=optional-service-key
⚠️ Important: Ensure JWT_SECRET is set in your environment before starting the KYC service. Without it, all authenticated endpoints will fail.

📊 KYC Verification Workflow

User Submission
    ↓
[KYCSubmission.tsx] → [kycApi.submitKYC()] → [Backend: POST /kyc-submit]
    ↓
Store in Database (status: 'pending')
    ↓
Admin Reviews
    ↓
[KYCAdminDashboard.tsx] → [kycApi.getPendingKYC()] → [Backend: GET /admin/kyc/pending]
    ↓
Admin Decision (Approve/Reject/Request Documents)
    ↓
[kycApi.approveKYC()|rejectKYC()|requestDocuments()]
    ↓
Update Database (status: 'approved'|'rejected'|'needs_review')
    ↓
User Checks Status
    ↓
[KYCStatus.tsx] → [kycApi.getKYCStatus()] → Display Result

📝 Database Schema

kyc_verification
├── id (UUID, PK)
├── userId (UUID, FK)
├── status (pending|approved|rejected|needs_review)
├── documentType (passport|license|national_id)
├── firstName, lastName, dateOfBirth, nationality
├── address (JSON)
├── submittedAt, reviewedAt, approvedAt
├── reviewedBy (FK to admin user)
└── rejectionReason

kyc_document_submission
├── id (UUID, PK)
├── kycVerificationId (UUID, FK)
├── documentType (front|back|selfie|proof_of_address)
├── documentUrl, fileName, fileSize, mimeType
└── verificationStatus (pending|approved|rejected)

kyc_verification_history
├── id (UUID, PK)
├── kycVerificationId (UUID, FK)
├── status
├── changeReason
├── changedBy, changedAt

❓ Troubleshooting

KYC Service Not Starting

  • Verify KYC_SERVICE_PORT is not already in use
  • Check DATABASE_URL_CORE is correct
  • Ensure JWT_SECRET is set in environment
  • Run: npm run build --workspace=@tripalfa/kyc-service

Authentication Errors (401)

  • Verify JWT token is present in Authorization header
  • Check token format: Bearer <token>
  • Ensure JWT_SECRET matches token issuer
  • Check token expiration

Permission Errors (403)

  • Verify user has admin role for admin endpoints
  • Check permissions: kyc:view, kyc:manage, kyc:approve
  • Admin dashboard requires kyc:manage permission
